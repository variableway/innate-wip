# 时尚消费品趋势追踪系统搭建指南

> 本文面向希望建立**可自动化、可持续、可量化**的时尚消费品数据追踪系统的个人用户。覆盖男性和女性消费品全品类，提供从"零代码"到"半自动化"的三种实现方案。

---

## 一、系统目标：你想追踪什么？

在搭建系统之前，先明确你要回答的问题：

| 问题类型 | 示例 | 数据源 |
|----------|------|--------|
| **热度趋势** | 哪个品牌最近在小红书被讨论最多？ | 小红书搜索指数、抖音话题量 |
| **价格趋势** | SK-II 神仙水过去一年价格走势如何？ | 什么值得买历史价格、中免日上 |
| **新品追踪** | Chanel 最近发布了什么新包/新香？ | 品牌官网、小红书新品笔记 |
| **二手市场** | Rolex 水鬼二手价格是涨是跌？ | Chrono24、得物、红布林 |
| **性别交叉** | 哪些香水正在"中性化"？男女都在买？ | 社交媒体讨论语义分析 |
| **季节周期** | 什么品类在双11/618/黑五折扣最大？ | 历年电商促销数据 |

**核心原则**：先定义问题，再选择工具。不要采集你不需要的数据。

## 三、三种实现方案（按投入成本排序）

### 方案 A：飞书多维表格 + 自动化插件（推荐入门，零代码）

**适合人群**：不想写代码，但希望有一定自动化能力的用户
**成本**：免费（飞书个人版）
**维护成本**：每周 30 分钟

#### 搭建步骤

**Step 1：创建基础数据表**

在飞书多维表格中创建以下表格：

**表 1：商品库（核心主表）**
```
字段：商品ID | 商品名称 | 品牌 | 品类 | 性别标签 | 层级 | 官方价格 | 当前最优价 | 二手参考价 | 数据来源 | 更新时间 | 趋势标签 | 备注
```

**表 2：价格追踪日志（时序数据）**
```
字段：记录ID | 商品ID(关联) | 日期 | 渠道 | 价格 | 折扣率 | 库存状态 | 抓取方式
```

**表 3：热度追踪（社交媒体）**
```
字段：日期 | 品牌 | 平台 | 搜索指数 | 话题量 | 笔记数 | 互动量 | 性别倾向 | 热门关联词
```

**表 4：新品/事件日历**
```
字段：日期 | 品牌 | 事件类型(新品发布/促销/代言人) | 事件描述 | 影响评估 | 相关商品
```

**Step 2：设置自动化规则**

飞书多维表格支持以下自动化（点击"自动化"→创建规则）：

| 触发条件 | 执行动作 | 用途 |
|----------|---------|------|
| 每天上午 9 点 | 发送提醒：手动更新今日价格 | 养成采集习惯 |
| 当"当前最优价"低于"历史最低价"的 90% | 发送飞书消息提醒 | 捕捉神价 |
| 当"趋势标签"变为"价格跳涨" | 发送预警消息 | 发现炒作/缺货 |
| 每周日晚 8 点 | 生成周报摘要（需配合脚本） | 定期复盘 |

**Step 3：接入外部数据源**

飞书多维表格支持以下集成：

- **网页剪辑器**：浏览淘宝/京东/什么值得买时，一键保存商品信息和价格到表格
- **API 连接器**（高级版）：可接入部分电商平台的公开 API
- **手动录入 + 快捷指令**：手机端快速录入专柜看到的信息

#### 优缺点

| 优点 | 缺点 |
|------|------|
| 完全零代码 | 自动化程度有限，大量数据仍需手动录入 |
| 移动端友好，随时记录 | 不支持复杂的数据分析和可视化 |
| 协作方便，可多人维护 | 免费版有行数限制（10000 行） |
| 模板丰富，上手快 | 无法自动抓取电商平台实时价格 |

**适合人群**：愿意写简单 Python 脚本，希望高度自定义的用户
**成本**：免费（Notion 个人版 + 本地运行 Python）
**维护成本**：初始搭建 4-6 小时，之后每周 15 分钟

```python
import requests
import sqlite3
from datetime import datetime
import json

def init_db():
    conn = sqlite3.connect('fashion_tracker.db')
    c = conn.cursor()
    c.execute('''
        CREATE TABLE IF NOT EXISTS price_history (
            id INTEGER PRIMARY KEY AUTOINCREMENT,
            product_id TEXT,
            product_name TEXT,
            brand TEXT,
            category TEXT,
            gender_tag TEXT,
            tier TEXT,
            channel TEXT,
            price REAL,
            original_price REAL,
            discount_rate REAL,
            date TEXT,
            url TEXT
        )
    ''')
    conn.commit()
    conn.close()

def record_price(product_info):
    """
    product_info = {
        'product_id': 'sk2_facial_treatment_230ml',
        'product_name': 'SK-II 护肤精华露 230ml',
        'brand': 'SK-II',
        'category': '护肤品',
        'gender_tag': '女性/通用',
        'tier': '高端',
        'channel': '中免日上',
        'price': 899.0,
        'original_price': 1540.0,
        'url': 'https://www.cdfgsanya.com/...'
    }
    """
    conn = sqlite3.connect('fashion_tracker.db')
    c = conn.cursor()
    
    discount_rate = product_info['price'] / product_info['original_price'] if product_info['original_price'] else 1.0
    
    c.execute('''
        INSERT INTO price_history 
        (product_id, product_name, brand, category, gender_tag, tier, channel, price, original_price, discount_rate, date, url)
        VALUES (?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?)
    ''', (
        product_info['product_id'],
        product_info['product_name'],
        product_info['brand'],
        product_info['category'],
        product_info['gender_tag'],
        product_info['tier'],
        product_info['channel'],
        product_info['price'],
        product_info['original_price'],
        discount_rate,
        datetime.now().isoformat(),
        product_info.get('url', '')
    ))
    
    conn.commit()
    conn.close()
    print(f"已记录: {product_info['product_name']} @ {product_info['channel']} = ¥{product_info['price']}")
```

**2. 趋势分析脚本**

```python
import sqlite3
import pandas as pd
import matplotlib.pyplot as plt
from datetime import datetime, timedelta

def analyze_price_trend(product_id, days=90):
    """分析某商品过去 N 天的价格趋势"""
    conn = sqlite3.connect('fashion_tracker.db')
    
    query = f"""
    SELECT date, channel, price, discount_rate 
    FROM price_history 
    WHERE product_id = '{product_id}' 
    AND date >= datetime('now', '-{days} days')
    ORDER BY date
    """
    
    df = pd.read_sql_query(query, conn)
    conn.close()
    
    if df.empty:
        print(f"无数据: {product_id}")
        return
    
    df['date'] = pd.to_datetime(df['date'])
    
    # 基础统计
    print(f"\n=== {product_id} 价格分析 ({days}天) ===")
    print(f"平均价格: ¥{df['price'].mean():.0f}")
    print(f"最低价格: ¥{df['price'].min():.0f}")
    print(f"最高价格: ¥{df['price'].max():.0f}")
    print(f"平均折扣: {df['discount_rate'].mean():.1%}")
    print(f"价格记录数: {len(df)}")
    
    # 渠道对比
    channel_stats = df.groupby('channel')['price'].agg(['mean', 'min', 'max', 'count'])
    print(f"\n按渠道统计:")
    print(channel_stats)
    
    # 绘制趋势图
    plt.figure(figsize=(12, 6))
    for channel in df['channel'].unique():
        channel_df = df[df['channel'] == channel]
        plt.plot(channel_df['date'], channel_df['price'], marker='o', label=channel, alpha=0.7)
    
    plt.title(f'{product_id} 价格趋势')
    plt.xlabel('日期')
    plt.ylabel('价格 (¥)')
    plt.legend()
    plt.xticks(rotation=45)
    plt.tight_layout()
    plt.savefig(f'{product_id}_trend.png', dpi=150)
    plt.close()
    print(f"\n趋势图已保存: {product_id}_trend.png")

if __name__ == '__main__':
    analyze_price_trend('dior_sauvage_edt_60ml', days=90)
```

```python
from notion_client import Client
import sqlite3

notion = Client(auth="secret_xxxxxxxxx")  # 从 Notion Integration 获取

def sync_to_notion():
    conn = sqlite3.connect('fashion_tracker.db')
    df = pd.read_sql_query("SELECT * FROM price_history ORDER BY date DESC LIMIT 100", conn)
    conn.close()
    
    database_id = "your-database-id"
    
    for _, row in df.iterrows():
        notion.pages.create(
            parent={"database_id": database_id},
            properties={
                "商品名称": {"title": [{"text": {"content": row['product_name']}}]},
                "品牌": {"select": {"name": row['brand']}},
                "品类": {"select": {"name": row['category']}},

                "日期": {"date": {"start": row['date']}},
            }
        )
```

```python
with tab1:
    st.header("实时价格监控")
    
    # 读取最新价格
    query = f"""
    SELECT ph.*, p.gender_tag, p.category
    FROM price_history ph
    JOIN products p ON ph.product_id = p.id
    WHERE p.gender_tag IN ({','.join(['?']*len(gender_filter))})
    AND p.category IN ({','.join(['?']*len(category_filter))})
    AND ph.date >= date('now', '-30 days')
```

## 四、两性交叉追踪的特殊维度

既然你希望同时追踪男性和女性消费品，系统需要特别设计**性别流动性和交叉分析**维度：

### 4.1 性别标签设计

不要简单地把商品标记为"男"或"女"，使用更丰富的标签系统：

```
主要性别标签：男性 / 女性 / 中性 / 未指定
使用场景标签：商务 / 休闲 / 运动 / 约会 / 礼品
性别流动性：传统男性向 / 传统女性向 / 中性化趋势 / 无性别
文化符号：硬汉 / 文艺 / 精致 / 极简 / 街头 / 商务精英
```

### 4.2 值得关注的两性交叉趋势

| 趋势现象 | 数据追踪方法 | 示例 |
|----------|-------------|------|
| **男性护肤品增长** | 追踪男性护肤品牌搜索量增速 vs 女性品牌 | 左颜右色、朗仕增速是否超过女性品牌？ |
| **中性香崛起** | 追踪被标记为"中性"的香水量和讨论热度 | Le Labo、Byredo 是否被男女同时讨论？ |
| **女性男香自用** | 小红书笔记语义分析："女生用男香" | 多少女性在买 Dior 旷野自用？ |
| **男性美妆入门** | 追踪男士素颜霜、眉笔、底妆搜索量 | 左颜右色素颜霜热度变化 |
| **包袋性别模糊** | 追踪男女同款包讨论量 | LV Keepall、Goyard 被男女同时讨论？ |
| **送礼反向流动** | 追踪"送男生/送女生"话题内容变化 | 男生开始收到护肤品礼盒？ |

如果你不想搭建复杂系统，以下是**今天就能启动的最小可行方案**：

|------|------|------|

### 每日 5 分钟流程

```

  └─ 打开小红书，搜索 1 个你关注的品牌，花 2 分钟看热榜笔记主题
  
中午休息:
  └─ 打开什么值得买，查看你收藏的商品是否有新低价，花 2 分钟

  └─ 如果有新发现，花 1 分钟录入到飞书/Notion
```

## 七、总结：选择适合你的方案

| 维度 | 方案 A：飞书表格 | 方案 B：Python+Notion | 方案 C：全栈Dashboard |
|------|----------------|---------------------|---------------------|
| **技术门槛** | 零代码 | 基础Python | 全栈开发 |
| **搭建时间** | 2小时 | 1-2天 | 1-2周 |
| **自动化程度** | 低（手动为主） | 中（半自动） | 高（全自动） |
| **数据隐私** | 飞书云端 | 本地+Notion云端 | 完全可控 |
| **最佳适用** | 刚入门P4，先养成记录习惯 | 已确认长期追踪需求 | 极客用户，多维度分析 |
| **我的建议** | **从这里开始** | 用1个月后升级到B | 3个月后再考虑 |

**最终建议**：

1. **今天**：下载飞书，创建第一个"商品追踪表"，录入你最近感兴趣的 5 个商品
2. **本周**：每天花 5 分钟查看价格/热度，养成数据敏感度
3. **本月**：如果坚持下来了，且数据量超过 100 条，考虑升级到方案 B
4. **长期**：系统不是目的，**用数据做出更好的消费决策和社交判断**才是目的

> 建立追踪系统的本质，是把你从"被动接收信息"变成"主动筛选信息"。当你开始用数据视角看待消费世界时，你会发现：那些看似复杂的时尚消费品，其实有清晰的规律可循。而你，正在成为那个能看见规律的人。
